
План действий при создании приложения
Шаг 1: Анализ требований
Определить функциональные и нефункциональные требования приложения.
Выделить основные роли пользователей: пользователь и администратор.
Разработать схему работы приложения, определив основные процессы: регистрация, авторизация, создание заявки, просмотр заявок, изменение статусов заявок.
Шаг 2: Проектирование базы данных
Определить сущности базы данных: пользователи, заявки.
Разработать схему базы данных, включающую в себя таблицы для хранения информации о пользователях и заявках.
Определить связи между таблицами.
Шаг 3: Разработка внутренней части 
Создания базы данных и таблиц.
Настроить сервер и базу данных 
Написать код регистрации и авторизации пользователей.
Реализовать возможность создания заявки и сохранения ее в базе данных.
Разработать функционал для просмотра заявок.
Шаг 4: Разработка внешней части
Создать пользовательский интерфейс для страниц регистрации, авторизации и создания заявки.
Разработать интерфейс для администратора, позволяющий просматривать и изменять статусы заявок.
Создать страницы для просмотра статистических данных по заявкам.
Шаг 5: Тестирование
Провести тестирование функционала регистрации, авторизации и создания заявок.
Проверить работу интерфейса на различных устройствах и браузерах.
Выявить и исправить возможные ошибки и уязвимости.
